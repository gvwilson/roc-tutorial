Roc in 100 Programs

A Gentle Introduction to Functional Programming

What This Is

  • Notes and working examples that instructors can use to perform a lesson
    • Do not expect novices with no prior Roc experience to learn from them on their own
  • Musical analogy
    • This is the chord changes and melody
    • We expect instructors to create an arrangement and/or improvise while delivering
  • Please see the license for terms of use, the Code of Conduct for community standards, and these guidelines for notes on contributing
  • Greg Wilson is a programmer, author, and educator based in Toronto

1: Hello Roc

  • Run roc repl at the command line
  • Character string evaluates to itself

src/interp-hello-roc.roc

"Hello, Roc"

out/interp-hello-roc.out

"Hello, Roc" : Str
  • The Str after the colon tells us the type of the value
  • Roc cares a lot about types

2: Numbers

src/interp-number-3.roc

3

out/interp-number-3.out

3 : Num *
  • 3 is a Num *
    • Num is short for “number”
    • * means this is a wildcard type
  • Compatible with more specific numeric types Int and Frac

3: Variables

  • Assigning a value to a name creates a variable
    • The value of the expression is automatically echoed

src/interp-var-def.roc

value = "high"

out/interp-var-def.out

"high" : Str
  • But all “variables” are actually constants

src/interp-var-redef.roc

value = "low"

out/interp-var-redef.out

thread 'main' panicked at 'not yet implemented', crates/compiler/mono/src/ir.rs:2742:23

4: Arithmetic

src/interp-arithmetic.roc

5 - 1 * 3 + 4 / 2

out/interp-arithmetic.out

4 : Frac *
  • Roc respects the usual precedence rules for arithmetic
  • Result is Frac because division creates a fractional number
    • And Num combined with Frac is Frac

6: Interpolation

    • Your responsibility to convert interpolated values to text

7: Defining Functions

  • Use \ followed by parameters (with no space after \)
  • Assign result to a name

src/interp-max3.roc

max3 = \a, b, c ->
    Num.max a (Num.max b c)
max3 5 2 7

out/interp-max3.out

<function> : Num a, Num a, Num a -> Num a

7 : Num *

8: First Application

src/first-app.roc

app "first"
    packages { cli: "https://github.com/roc-lang/basic-cli/releases/download/0.8.1/x8URkvfyi9I0QhmVG98roKBUs_AZRkLFwFJVJ3942YA.tar.br" }
    imports [cli.Stdout]
    provides [main] to cli

main =
    Stdout.line "First application"

out/first-app.out

First application
  • Downloads the basic-cli platform the first time it runs
  • Uses the cached version after that
  • We’ll go through the details but:
    • Create an application called first
    • Get the basic-cli platform as a package called cli
    • Get Stdout (standard output) from that package
    • Provide our main function to cli
    • Define the main we’re providing
  • Yeah, it feels a bit like Java’s public static void main
